# A-0 Мультимодальный анализ эмоционального состояния в HR-процессах

На вход должно быть то что у нас хранится в бд
# Входы:
* Данные конфигурации анализа
* Пользовательский запрос (Создание сессии, конфигов)
* Список участников

# Управление:
* 152-ФЗ, 149-ФЗ
* Методики HR-анализа
* Алгоритмы детекции
* Модели и алгоритмы анализа эмоций
* форма отчета

# Механизмы:
* Веб-камера, микрофон (объеденены)
* Сервер
* Клиент

# Выходы:
* Аналитический отчет

# Описание 
## Проведение сессии(А-1):
 Клиет создает конфиг, делает запрос на создание сессии и приглашает участников
### Входы:
* Пользовательский запрос (Создание сессии, конфигов)
* Список участников(нужно ли если можно через инвайт ссылку сделать)

### Управление:
Закон о перс. данных, Согласие на захват медиа, Закон о хранении данных
### Механизмы:
* Видеопоток
* Аудиопоток
* Сервер
* Пользователь

### Выходы:
* Видеопоток
* Аудиопоток

## Обработка данны(А-2):
 Собранные данные проходят все нужные этапы обработки, извне применяются алгоритмы детекции 

### Входы:
* Видеопоток
* Аудиопоток
* Данные конфигурации анализа

### Управление:
* Алгоритмы детекции

### Механизмы:
* Сервер

### Выходы: 
* Признаки видео
* Признаки аудио

## Анализ данных(А-3):
 Предобученные нейронки дают argmax эмоций для данных и модули
### Входы:
* Признаки видео
* Признаки аудио
* Данные конфигурации анализа
### Управление:
* Модели и алгоритмы анализа эмоций
### Механизмы:
* Сервер
### Выходы:
* Результаты анализ

 ## Интерпретация данных и формирование отчета(А-4):
 Формируется отчет по выбранной форме отчета

### Входы:
* Результаты анализа
* Данные конфигурации анализа

### Управление:
* Сервер
* Клиент
### Механизмы:
* форма отчета
* Правила интерпретации
### Выходы: 
* Аналитический отчет
